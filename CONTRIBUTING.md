# Contributing to Visco

Thank you for your interest in improving Visco. Contributions that improve the
learning experience, visualizations, documentation, accessibility, or code
quality are welcome.

## License and fork policy

Visco is source-available under the [Visco Source Available License v1.0](LICENSE),
not an open-source license. You may fork this repository for development,
learning, and preparing or submitting contributions to the official Visco
repository.

Forks are only permitted for those contribution-related purposes. Do not
publish, redistribute, commercialize, or present your fork or a modified
version as a separate project. Do not use the code to create a competing or
derivative product without prior written permission from Saheli Mondal.

## Contribution workflow

1. Fork the official Visco repository on GitHub.
2. Create a branch in your fork for one focused change.

   ```bash
   git checkout -b feat/short-description
   ```

3. Make the change while preserving existing copyright and license notices.
4. Run the relevant checks locally:

   ```bash
   npm run lint
   npm run build
   ```

5. Commit with a clear, descriptive message.
6. Open a pull request from your branch to the official Visco repository.
7. Describe the problem, the solution, and any testing completed in the pull
   request.

Please keep pull requests focused and avoid unrelated formatting or generated
file changes. By submitting a pull request, you confirm that your contribution
may be used in the official Visco repository under its license.
